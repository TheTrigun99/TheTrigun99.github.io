Introduction

Word2Vec apprend une représentation dense de chaque mot en observant son contexte. Des mots qui apparaissent dans des contextes similaires obtiennent des vecteurs proches. Ces vecteurs servent ensuite à mesurer des proximités, faire des analogies, ou initialiser un modèle de recherche sémantique.

Intuition : CBOW vs Skip-gram

Deux variantes symétriques :

  • CBOW (Continuous Bag of Words) prédit un mot à partir du contexte. Plus rapide, efficace sur grands corpus.
  • Skip-gram prédit le contexte à partir d'un mot. Plus précis sur petits corpus et mots rares.

Je pars souvent sur Skip-gram (sg=1) avec une fenêtre modeste (5 à 8) pour capter les proximités locales.

Traitement du corpus

Objectif : un texte homogène et propre avant l'apprentissage.

  • Minuscules, retrait de ponctuation inutile, suppression des doublons et lignes vides.
  • min_count pour filtrer les mots trop rares.
  • Tokenisation simple : gensim.utils.simple_preprocess suffit dans beaucoup de cas.

Entraînement & hyperparamètres

Configuration de base :

  • vector_size : 100 à 300 (200 par défaut ici).
  • window : 5 pour du texte généraliste.
  • min_count : 3 pour filtrer le très rare.
  • sg : 1 (Skip-gram) ou 0 (CBOW).
  • epochs : monte si le corpus est modeste.

Surveille le temps d'entraînement (nombre de threads via workers) et la stabilité du vocabulaire (pas trop de mots éliminés).

Evaluation rapide

  • Similarités : mots proches/éloignés pour voir si les contextes sont cohérents.
  • Analogies : roi - homme + femme ≈ reine ou des exemples pertinents à ton domaine.
  • OOV : mots hors vocabulaire. Ajuste min_count ou enrichis le corpus.

Usage pratique

  • Recherche de similarité : vecteurs moyens de documents, cosine.
  • Initialisation d'un moteur vectoriel (FAISS, etc.).
  • Features pour des modèles classiques (clustering, classification).

Bloc code (Python + Gensim)

import re
from pathlib import Path
from gensim.models import Word2Vec
from gensim.utils import simple_preprocess

raw_text = Path("corpus.txt").read_text(encoding="utf-8")

def clean_sentence(sent):
    sent = sent.lower()
    sent = re.sub(r"[^a-zàâçéèêëîïôûùüÿñæœ\\s]", " ", sent)
    return sent

sentences = [
    simple_preprocess(clean_sentence(s))
    for s in raw_text.split("\\n")
    if s.strip()
]

model = Word2Vec(
    sentences=sentences,
    vector_size=200,
    window=5,
    min_count=3,
    workers=4,
    sg=1,   # Skip-gram (0 pour CBOW)
    epochs=12,
)

print(model.wv.most_similar("nlp", topn=5))

model.save("word2vec.model")
model.wv.save_word2vec_format("word2vec.vec")

Adapte la regex à ton alphabet, ajuste min_count si le corpus est petit, et modifie window selon la granularité voulue.
