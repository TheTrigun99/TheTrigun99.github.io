Word2Vec expliqué en pratique

Comment passer d'un corpus brut à des vecteurs de mots exploitables pour la recherche de similarité ou l'analyse de proximité sémantique.

Recette rapide Voir le code

Objectifs du tuto

  • Comprendre CBOW vs Skip-gram
  • Nettoyer et tokeniser un corpus
  • Former et sauvegarder un modèle
  • Tester la similarité entre mots

Recette rapide

Checklist
1. Corpus

Rassembler des textes homogènes, retirer le bruit.

2. Nettoyage

Lowercase, retirer les stopwords, garder les mots utiles.

3. Entraînement

CBOW (rapide) ou Skip-gram (précis) selon la taille du corpus.

4. Évaluation

Tester la similarité, vérifier les mots hors vocabulaire.

5. Sauvegarde

Exporter le modèle et les vecteurs pour réutilisation.

Pourquoi Word2Vec ?

Chaque mot est représenté par un vecteur dense qui capture le contexte d'apparition. On peut alors mesurer la proximité sémantique avec des distances de cosine, alimenter un moteur de recherche ou un système de recommandations textuelles.

Choisir le mode

CBOW prédit un mot à partir du contexte : plus rapide, efficace sur grands corpus. Skip-gram prédit le contexte à partir d'un mot : plus fin sur petits corpus, capte mieux les mots rares.

Paramètres clés

  • vector_size : 100-300 pour commencer
  • window : 5-8 pour du texte généraliste
  • min_count : filtrer les mots trop rares
  • sg : 0 = CBOW, 1 = Skip-gram

Exemple de code (Python + Gensim)

Copier / coller 
import re
from pathlib import Path
from gensim.models import Word2Vec
from gensim.utils import simple_preprocess

# 1) Charger et préparer les données
raw_text = Path("corpus.txt").read_text(encoding="utf-8")

def clean_sentence(sent):
    sent = sent.lower()
    sent = re.sub(r"[^a-zàâçéèêëîïôûùüÿñæœ\\s]", " ", sent)
    return sent

sentences = [simple_preprocess(clean_sentence(s)) for s in raw_text.split("\\n") if s.strip()]

# 2) Entraîner le modèle
model = Word2Vec(
    sentences=sentences,
    vector_size=200,
    window=5,
    min_count=3,
    workers=4,
    sg=1,  # 1 = Skip-gram, 0 = CBOW
    epochs=12,
)

# 3) Inspecter quelques résultats
print(model.wv.most_similar("nlp", topn=5))

# 4) Sauvegarder
model.save("word2vec.model")
model.wv.save_word2vec_format("word2vec.vec")

Pense à adapter la regex pour les caractères spécifiques à ta langue et à ajuster min_count si ton corpus est petit.

Évaluer rapidement

  • Comparer des paires de mots proches / éloignés
  • Tester l'analogie : roi - homme + femme ≈ reine
  • Repérer les mots hors vocabulaire (OOV)

Astuces pratiques

  • Tokenisation cohérente : tout en minuscule ou pas
  • Retirer les chiffres si non pertinents
  • Essayer plusieurs tailles de fenêtre (3, 5, 8)
  • Limiter les phrases trop courtes : elles apprennent peu
