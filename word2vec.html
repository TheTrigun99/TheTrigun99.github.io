Introduction Traitement Modèle

Page 1 · Introduction

Pourquoi Word2Vec ? Capturer la proximité sémantique en apprenant des vecteurs denses de mots. Utile pour la recherche, les recommandations et la comparaison de documents.

Deux variantes : CBOW (rapide, préférée sur grands corpus) et Skip-gram (plus précis sur petit corpus, sensible aux mots rares). Je documente ici l'approche que j'utilise sur mes propres données.

Page 2 · Traitement

Corpus homogène : articles, docs internes ou notes. Nettoyage de base : passage en minuscules, retrait de ponctuation non utile, suppression des mots trop rares (min_count).

Tokenisation simple avec gensim.utils.simple_preprocess. Pense à filtrer les lignes vides et à adapter la regex si le corpus contient des caractères spécifiques.

Page 3 · Modèle

Paramètres de départ : vector_size=200, window=5, min_count=3, sg=1 pour Skip-gram, epochs=12. Ajuste selon la taille du corpus.

Évaluer rapidement avec quelques analogies ou similarités connues, surveiller les OOV (mots absents du vocabulaire). Exporter .model et .vec pour réutilisation.

Exemple rapide (Python + Gensim)

import re
from pathlib import Path
from gensim.models import Word2Vec
from gensim.utils import simple_preprocess

raw_text = Path("corpus.txt").read_text(encoding="utf-8")

def clean_sentence(sent):
    sent = sent.lower()
    sent = re.sub(r"[^a-zàâçéèêëîïôûùüÿñæœ\\s]", " ", sent)
    return sent

sentences = [
    simple_preprocess(clean_sentence(s))
    for s in raw_text.split("\\n")
    if s.strip()
]

model = Word2Vec(
    sentences=sentences,
    vector_size=200,
    window=5,
    min_count=3,
    workers=4,
    sg=1,   # 1 = Skip-gram, 0 = CBOW
    epochs=12,
)

print(model.wv.most_similar("nlp", topn=5))

model.save("word2vec.model")
model.wv.save_word2vec_format("word2vec.vec")

Adapte la regex à ta langue, ajuste min_count si le corpus est petit. Tout ce bloc est prêt à remplacer par tes propres notes.
