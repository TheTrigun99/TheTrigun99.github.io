Projet Word2Vec

Comprendre et construire mes embeddings

Un post long pour expliquer ce que je fais avec Word2Vec : du corpus brut aux vecteurs utilisables en recherche de similarité.

Aller au post Voir le code

1. Idée

Word2Vec produit des représentations denses de mots en se basant sur leur contexte. L'idée : capter la proximité sémantique pour améliorer la recherche et les recommandations.

2. Corpus et nettoyage

Je pars d'un corpus homogène (articles, docs internes ou notes). Je nettoie : minuscules, retrait de la ponctuation inutile, suppression des mots trop rares. Le but est de garder uniquement ce qui porte du sens.

3. Entraînement

Je choisis CBOW (rapide) ou Skip-gram (plus précis sur petit corpus). Paramètres de base : vector_size=200, window=5, min_count=3. J'augmente epochs si le corpus est modeste.

4. Évaluation simple

Je teste quelques analogies et paires de mots proches/éloignés. Je vérifie les mots hors vocabulaire (OOV) pour ajuster min_count.

5. Export

Je sauvegarde le modèle (.model) et les vecteurs (.vec) pour les réutiliser dans un service ou un notebook.

Exemple rapide (Python + Gensim)

import re
from pathlib import Path
from gensim.models import Word2Vec
from gensim.utils import simple_preprocess

raw_text = Path("corpus.txt").read_text(encoding="utf-8")

def clean_sentence(sent):
    sent = sent.lower()
    sent = re.sub(r"[^a-zàâçéèêëîïôûùüÿñæœ\\s]", " ", sent)
    return sent

sentences = [
    simple_preprocess(clean_sentence(s))
    for s in raw_text.split("\\n")
    if s.strip()
]

model = Word2Vec(
    sentences=sentences,
    vector_size=200,
    window=5,
    min_count=3,
    workers=4,
    sg=1,   # 1 = Skip-gram, 0 = CBOW
    epochs=12,
)

print(model.wv.most_similar("nlp", topn=5))

model.save("word2vec.model")
model.wv.save_word2vec_format("word2vec.vec")

Adapte la regex à ta langue, ajuste min_count si le corpus est petit. Tout ce bloc est prêt à remplacer par tes propres notes.
